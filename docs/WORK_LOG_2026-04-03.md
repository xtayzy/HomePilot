# Журнал работ — 3 апреля 2026

Краткий перечень изменений в репозитории HomePilot (незакоммиченное состояние на дату записи), чтобы перенести формулировки в Jira и Confluence.

---

## Резюме для Confluence (1 абзац)

Реализована **админ-панель** (роли `admin` и `support`): бэкенд-модуль `/admin` со статистикой, списками пользователей, подписок, визитов, исполнителей, тикетов поддержки и платежей, а также операциями (назначение исполнителя, правки подписки/визита, ответы в поддержке, приглашения исполнителей). На фронте добавлены **маршруты `/admin/*`**, layout с навигацией и страницы обзора и деталей. Добавлена **регистрация исполнителя по инвайту** (`/register-executor`). Улучшены **доступы к визитам** для staff и исправлена **сериализация подписок** в async SQLAlchemy (без ленивой подгрузки `sub.tariff`). Вход перенаправляет admin/support в админку; layout клиента и исполнителя **перенаправляют** staff в `/admin`. В seed добавлен **тестовый администратор**. Параллельно подготовлен **пакет документации** (Confluence, безопасность, бэклог Jira).

## Backend

| Тема | Что сделано |
|------|-------------|
| Admin API | Новый роутер `prefix=/admin`: статистика, пользователи (список, карточка с подписками/визитами/платежами/тикетами), подписки (список, деталь, patch), визиты (список, patch, назначение исполнителя), исполнители (список, статус, создание invite), поддержка (тикеты, сообщения, статус), платежи (список). Разделение прав: staff (`admin` + `support`) vs только `admin` где нужно. |
| Роутер v1 | Подключён `admin` в `api_v1_router`. |
| Схемы | Новый модуль `app/schemas/admin.py` для тел запросов админки. |
| Подписки (публичный API) | `_subscription_to_response` принимает `tariff` и `apartment_type` явно; устранён риск `MissingGreenlet` при доступе к связям после async-операций. |
| Визиты (публичный API) | `get_visit`: для admin/support доступ без фильтра по `user_id`; подгрузка `subscription.user`. `get_visit_photo`: доступ для admin/support. |
| Сервис визитов | `reschedule_visit`: флаг `skip_owner_check` для сценариев админки (перенос от имени системы без проверки владельца). |
| Seed | Функция `seed_admin`: создаётся пользователь-администратор (email/пароль заданы константами в `seed.py` — для продакшена заменить/убрать). |

## Frontend

| Тема | Что сделано |
|------|-------------|
| Маршрутизация | В `App.tsx`: вложенные маршруты под `AdminLayout` — обзор, пользователи, подписки, визиты, исполнители, поддержка, платежи; маршрут `/register-executor`. |
| API-клиент | Расширение `client.ts`: типы и функции для всех основных admin-эндпоинтов + `registerExecutor`. |
| UI админки | Компонент `AdminLayout` (sidebar, защита по роли). Страницы в `pages/admin/`: `AdminOverview`, `AdminUsers`, `AdminUserDetail`, `AdminSubscriptions`, `AdminSubscriptionDetail`, `AdminVisits`, `AdminVisitDetail`, `AdminExecutors`, `AdminSupport`, `AdminTicketDetail`, `AdminPayments`. |
| Регистрация исполнителя | Страница `RegisterExecutorPage`: форма с кодом приглашения, вход по токенам и редирект в `/executor`. |
| Вход и роли | `LoginPage`: после логина редирект admin/support → `/admin`, executor → `/executor`, иначе → `/dashboard`. |
| Защита зон | `DashboardLayout` и `ExecutorLayout`: пользователи admin/support перенаправляются в `/admin`. |
